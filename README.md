# UniHub Web — المرحلة 1

موقع Next.js 15 / React 19 / TypeScript / Tailwind يعيد بناء واجهة تطبيق UniHub
(Flutter) على نفس مشروع Firebase الحالي (`my-university-cc6bc`) بدون أي تغيير
في الـ Collections أو الـ Security Rules أو منطق العمل.

## ✅ ما تم إنجازه (المرحلة 1 + المرحلة 2)

- **البنية الأساسية**: Next.js 15 App Router + TypeScript + Tailwind، جاهزة للتشغيل والبناء (تم اختبار `npm run build` بنجاح).
- **Firebase**: نفس المشروع، نفس الإعدادات (`lib/firebase/client.ts`)، بما فيها persistence مطابق لإعدادات Flutter.
- **Supabase Storage**: نفس الـ buckets المستخدمة في تطبيق Flutter للملفات والبانرات (`lib/firebase/storage.ts`).
- **طبقة الأنواع (Types)**: `lib/types/models.ts` تطابق حرفيًا `models.dart`.
- **الثوابت الأكاديمية**: `lib/constants/academic.ts` — نفس قائمة الكليات/الأقسام/تجاوزات عدد المراحل حرفيًا.
- **المصادقة**: `lib/hooks/useAuth.tsx` — يطابق `login`/`signUp`/`_tryLoadInstructor`/`_loadOrCreateProfile`.
- **Realtime Sync**: كل الشاشات تستخدم `onSnapshot` — أي تغيير من Flutter يظهر فورًا بالموقع والعكس.
- **Design System + Layout**: حسب المواصفات بالكامل (`tailwind.config.ts`, `app/globals.css`, `components/layout/*`).
- **شاشات مكتملة بمنطق مطابق 100% لـ Flutter**:
  - `/login` — تسجيل الدخول/إنشاء حساب مع التوجيه الصحيح.
  - `/home` — لوحة التحكم (بانرات، مواد، إعلانات).
  - `/schedule` — الجدول الأسبوعي + الواجبات + الامتحانات.
  - `/results` — الدرجات لكل مادة (بنفس معادلة `GradeSheet.totalFor`).
  - `/search` — بحث فوري بالمواد والملفات (نفس منطق warm-up في `search_screen.dart`).
  - `/subjects/[id]` — تفاصيل المادة: نسبة الحضور، ملخص الدرجات، تبويبات تصنيف الملفات، قائمة الملفات.
  - `/files/[id]` — تفاصيل الملف: المعاينة، التعليقات، صلاحية الحذف (لصاحب الملف أو الأدمن فقط)، رابط المساعد الذكي.

## ✅ ما تم إنجازه (المرحلة 1 + 2 + 3)

جميع نقاط المرحلة 1 و2 (راجع تاريخ المحادثة)، بالإضافة إلى:

- **`/notifications`** — قائمة الإشعارات (جلب لمرة واحدة بحد 30، مطابق لـ `_loadContentForUser`) + "تحديد الكل كمقروء" (batch write مطابق لـ `markAllNotificationsRead`).
- **`/announcements`** — نفس شاشة `/home` إعلانات لكن بصفحة كاملة + نافذة "نشر إعلان" لممثل الشعبة فقط (`isClassRep`)، بنفس نطاقي "شعبتي فقط" / "كل المرحلة".
- **`/profile`** — لوحة الحساب: الإحصائيات (ملفات مرفوعة/نقاط/محفوظات)، شارة "ممثل الشعبة"/"سوبر أدمن"/"أدمن قسم"، وقائمة الإعدادات.
- **`/profile/edit`** — تعديل الاسم (`updateAccountName`).
- **`/profile/notifications`** — تفعيل/تعطيل 3 أنواع إشعارات (`updateNotificationPreferences`)، تُحفظ فورًا بـ Firestore.
- **`/profile/followed`** — المواد المتابَعة + إلغاء المتابعة (`followedSubjects` / `toggleFollowSubject`، بما فيها تقسيم الطلب كل 10 IDs كما بـ Dart).
- **`/onboarding`** — كلية → قسم → مرحلة → شعبة → نوع الدراسة، يحفظ عبر `completeOnboarding`.
  - **⚠️ تبسيط واعٍ**: شاشة Flutter الأصلية تبدأ أولاً بتحديد المحافظة (GPS) واختيار الجامعة قبل الكلية. لم يتم بناء هذا الجزء بعد بالموقع لتوفير الوقت، والحقول `provinceId`/`universityId` تُحفظ فاضية مؤقتًا.
- **`/ai-assistant`** — محرك المطابقة (TF-IDF Retrieval) منقول **حرفيًا بالكامل** من `ai_assistant_engine.dart`، يعرض تبويبات "اسألني"/"الملخص"/"أسئلة امتحانية" من مستند `ai_content/{fileId}` بـ Firestore.
  - **⚠️ تبسيط واعٍ**: التوليد المحلي التلقائي من ملف PDF لأول مرة (`local_ai_generator.dart`) لم يُبنَ بعد بالموقع (يحتاج مكتبة تحليل PDF بالمتصفح).

## ✅ المرحلة 4: شاشات المحاضر (Instructor) الكاملة

- **`/instructor/signup`** — تسجيل حساب دكتور: كلية → قسم → اختيار مواده حسب المرحلة، بحالة `pending` بانتظار موافقة سوبر أدمن (`instructorSignUp`).
  - **⚠️ تبسيط واعٍ**: تستخدم قوائم الكليات/الأقسام العامة (`lib/constants/academic.ts`) بدل بيانات `iraq_locations.dart` الحقيقية.
- **`/instructor/pending`** — شاشة الانتظار + "تحقق الآن" (`refreshInstructorStatus`).
- **`/instructor/home`** — لوحة الدكتور: مواده المعتمدة + روابط الحضور/الدرجات.
- **`/instructor/attendance`** — أخذ الحضور (`rosterFor` + `saveAttendanceSession`، id ثابت يعيد فتح نفس الجلسة عند التعديل).
- **`/instructor/grades`** — إدخال الدرجات (نظام كورس/فصلي، فتح الورقة المحفوظة تلقائيًا، `saveGradeSheet`).
- **`/instructor/grades/audit`** — سجل تدقيق التعديلات (`auditLogForSheet`).

## ✅ المرحلة 5: شاشات الأدمن

- **`/admin/subjects`** — إضافة/حذف مواد (`createSubject`/`deleteSubject`)، سوبر أدمن يختار أي كلية/قسم، أدمن القسم محصور بقسمه.
- **`/admin/management`** — 3 تبويبات: الأدمن (بحث بالإيميل + ترقية طالب لأدمن قسم بنطاق دقيق stage/section/study_type، إلغاء الصلاحية)، ممثلو الشعب (تعيين/إلغاء)، طلبات الدكاترة (اعتماد/رفض).
- **`/admin/banners`** — رفع/تعطيل/حذف الصور الإعلانية (Supabase Storage)، على مستوى الجامعة أو قسم محدد.
- **`/admin/broadcast`** — نشر تبليغ فوري/إعلان/امتحان بنطاق مخصص (كلية/قسم/مرحلة/شعبة/نوع دراسة)، أو بث عام لسوبر أدمن فقط.
  - **⚠️ تبسيط واعٍ**: نموذج "تأجيل امتحان موجود" (postponeExam) من الشاشة الأصلية لم يُبنَ بعد — فقط إضافة امتحان جديد.

## ✅ المرحلة 6 (الأخيرة): آخر 3 نواقص

- **`/instructor/export`** — تصدير تقرير Excel حقيقي (درجات أو حضور) لمادة+شعبة+نوع دراسة، بنفس صيغة الأعمدة تمامًا (باستخدام مكتبة `xlsx` بدل حزمة `excel` في Dart)، يُنزَّل الملف مباشرة على جهاز المستخدم.
- **المساعد الذكي — توليد تلقائي من PDF بالمتصفح**: تم نقل `local_ai_generator.dart` بالكامل (تقسيم فقرات → TF-IDF → تلخيص استخراجي بالـ centroid → توليد أسئلة بقواعد نصية) حرفيًا إلى `lib/services/localAiGenerator.ts`، مع استخراج نص PDF بمكتبة `pdf.js`. الآن: أي طالب يفتح ملف محاضرة PDF ما عنده محتوى AI جاهز، يتحلل تلقائيًا على جهازه (نفس سلوك Flutter بالضبط)، وممثل الشعبة/الأدمن يقدر يحفظه لكل الطلاب بزر واحد بدل رفع ملف JSON يدويًا من سكربت بايثون خارجي (تبسيط للمسار، نفس النتيجة النهائية بـ Firestore).
- **تأجيل امتحان موجود** — أُضيف لصفحة `/admin/broadcast` (تبويب امتحان): عرض الامتحانات غير المؤجّلة بنفس النطاق، وتأجيل أي منها بتاريخ جديد/ملاحظة.

## 📱 PWA (تطبيق ويب قابل للتثبيت)

- **`public/manifest.json`** — اسم، أيقونات، ألوان الهوية، `display: standalone`، RTL/عربي.
- **`public/icons/`** — أيقونات مولّدة بتدرج الهوية (بنفسجي→أزرق) وحرف U، بكل المقاسات المطلوبة.
- **`public/sw.js`** — Service Worker بسيط: شرط التثبيت + تخزين مؤقت لغلاف الموقع (لا يتدخل بطلبات Firebase/Firestore/Supabase إطلاقًا).
- **`components/install-prompt.tsx`** — بانر تثبيت تلقائي (Chrome/Android + تلميح خاص لآيفون/سفاري).
- **يعمل فقط على HTTPS** (أو `localhost` محليًا) — متطلب أساسي من المتصفحات لأي Service Worker.

بعد رفع الموقع على أي استضافة HTTPS (Vercel مثلاً)، يقدر أي طالب يضغط "تثبيت" ويشتغل الموقع من شاشته الرئيسية بواجهة تطبيق كاملة.

## 🚧 المتبقي (تبسيطات موثّقة فقط، لا نواقص وظيفية جوهرية)

- onboarding وتسجيل الدكتور يستخدمان قوائم كليات/أقسام عامة بدل بيانات `iraq_locations.dart` الحقيقية (محافظة → جامعة عبر GPS).
- استخراج نص PDF بالمتصفح لا يدعم ملفات صور ماسحة ضوئية بدون طبقة نص (نفس قيد Flutter الأصلي فعليًا).

بهذا، **كل شاشة وكل وظيفة رئيسية بمشروع Flutter موجودة الآن بالموقع** بمنطق مطابق تم التحقق منه من الكود الأصلي مباشرة، وليس تخمينًا.

## ✨ ميزة جديدة (غير موجودة أصلاً بتطبيق Flutter، أُضيفت بطلب صريح)

**طلب تدريس مواد إضافية لدكتور معتمد مسبقًا:**
- بصفحة `/instructor/home` زر "+ إضافة مادة" يفتح قائمة مواد قسمه (عدا مواده المعتمدة/المطلوبة أصلاً)، يختار منها ويرسل طلب.
- المادة الجديدة تدخل بحالة "بانتظار موافقة الأدمن" وتظهر بقسم خاص بنفس الصفحة، بدون أي تأثير على مواده المعتمدة الحالية أو حالة حسابه.
- بصفحة `/admin/management` (تبويب "طلبات الدكاترة") قسم جديد "طلبات مواد إضافية من دكاترة معتمدين" يعتمد أو يرفض الطلب الجديد فقط.
- **ملاحظة**: هذه ميزة أضفتها بناءً على طلبك المباشر ولا وجود لها بكود Flutter الأصلي — البنية التحتية (حقول `requested_subject_ids`/`approved_subject_ids` بمستند `instructors`) كانت موجودة أصلاً وتسمح بهذا دون تعديل أي شيء بقاعدة البيانات.

## ✅ إعادة بناء `/schedule` كـ"الجدول الذكي"

كانت الصفحة تعرض الجدول مجمّعًا حسب كل الأيام مرة وحدة، بدون إدارة. أعدت بناءها بمطابقة كاملة لـ `schedule_screen.dart`:
- العنوان "الجدول الذكي" بنفس تدرج الألوان، وشريط بحث فعلي (يفلتر بالاسم/القاعة/الدكتور، وليس شكليًا بس مثل Flutter).
- شريط تبويب أيام الأسبوع (الأحد → السبت) يعرض جدول يوم واحد بالمرة (نفس UX الأصلي بالضبط).
- **إدارة كاملة** لممثل الشعبة وأي أدمن (`canManage`): زر "+" عائم يفتح قائمة (إضافة محاضرة/واجب/امتحان)، مع نفس منطق `_resolveScope` بالضبط (نطاق تلقائي لممثل الشعبة أو أدمن قسم محدد النطاق بالكامل، وإلا نافذة اختيار كلية/قسم/مرحلة/شعبة/نوع دراسة يدويًا لسوبر أدمن/أدمن بنطاق عام).
- زر حذف على كل محاضرة/واجب/امتحان لمن يملك صلاحية الإدارة.

## ملاحظة عن Supabase

قيم `.env.local` الخاصة بـ Supabase منقولة فعليًا من `supabase_config.dart` في مشروع Flutter (نفس الـ buckets: `files`, `promo_banners`) — لا حاجة لإنشاء مشروع جديد.

## ملاحظة أمنية

مفاتيح Firebase الموجودة في `.env.local` هي Web API Keys وهي ليست سرّية بطبيعتها (Firebase تعتمد على Security Rules للحماية وليس إخفاء المفتاح) — لكنها موضوعة في env vars كأفضل ممارسة. لم يتم تغيير أي Security Rule.
